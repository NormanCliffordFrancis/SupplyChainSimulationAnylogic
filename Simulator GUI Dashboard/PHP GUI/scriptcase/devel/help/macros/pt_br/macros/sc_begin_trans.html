sc_begin_trans("Conexão")

Essa macro inicia um conjunto de transações no banco de dados.

Nas aplicações de fomulário, os eventos que essa macro pode rodar dependem da atualização da base de dados (onAfterInsert, onAfterUpdate, onAfterDelete, onBeforeInsert, onBeforeUpdate or onBeforeDelete) são automaticamente é protegida por um controle de transação desde que a conexão seja a mesma da aplicação.

Em outros casos, se o usuário deseja fazer um controle de transação, ele precisa iniciar essa macro e finalizar com "sc_commit_trans()" macro para confirar a atualização ou o "sc_rollback_trans" para cancelar a transação.

O parâmetro "conexão" é opicional, necessário apenas se o comando é executado numa base de diferente da especificada na aplicação.
