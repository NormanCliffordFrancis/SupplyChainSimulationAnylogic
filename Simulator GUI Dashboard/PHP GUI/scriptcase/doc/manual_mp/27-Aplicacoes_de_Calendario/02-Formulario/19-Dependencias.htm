DEPENDENCIES

 

A resource to allow to link related tables, defining a dependency between tables. For example, the table of orders and items, removing a record from the table of orders, all its items also have to be removed. See an example using this resource below.

 

 

 

1º- First define a new dependence. In this example we will create an dependency between adm_suppliers_products (Dependent table) with the table adm_supplier (Actual application). Click on the button new dependency to begin.

 

2º - Select the dependent table and inform the amount of fields related to the father and child table. In this example only one field is related : the supplier code.

 

3º - Select the keys (linking fields) : adm_suppliers -> supplierID - adm_suppliers_products ->supplierID.


4º - Click on the button generate Scripts to define the dependence rule.
