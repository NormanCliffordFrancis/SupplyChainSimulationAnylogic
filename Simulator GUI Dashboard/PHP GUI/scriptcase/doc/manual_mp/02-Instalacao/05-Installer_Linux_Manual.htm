Installer Manual Scriptcase - Linux

 

This article will explain how to install and configure a PHP 5.4 and Apache 2.2 environment to receive the Scriptcase.

Follow the steps below very carefully.

Note: This article was based on Debian Wheezy distribution

 

 

Preparing the environment to receive the PHP 5.4 and Apache 2.2



First, edit the sources.list file:

sudo nano /etc/apt/sources.list



Add the following line to the end of the file:

deb-src http://packages.dotdeb.org wheezy all



Update your system using the following command:

sudo apt-get update


 

Installing PHP 5.4

 

Use the following command to install PHP 5.4

sudo apt-get install php5 php5-dev



Accept the installation of the dependencies.


Check the PHP version with the command:

php -v

 

Important: By default PHP disables some functions that are important for Scriptcase. So go to your php.ini (/etc/php5/apache2/php.ini) and change the line of disable_functions variable as shown below:

 


To enable the databases, we selected some examples of installation command:

MSSQL Server use the command sudo apt-get install php5-mssql



PostgreSQL use the command sudo apt-get install php5-pgsql



MySQL use the command sudo apt-get install php5-mysql



SQLite use the command sudo apt-get install php5-sqlite



Firebird use the command sudo apt-get install php5-interbase



Odbc use the command sudo apt-get install php5-odbc

 

 

Enabling GD Library

 

Use the command sudo apt-get install php5-gd to enable GD library.

Accept the installation of the dependencies.

 

 

Enabling Zend Guard Loader

 

Download Zend Guard Loader for Linux, the link below and place it on /usr/lib/php5/Zend.

Download ZendGuardLoader

 

Set recursive 777 permissions on the folder /usr/lib/php5/Zend

sudo chmod 777 -R /usr/lib/php5/Zend

 

Open php.ini (/etc/php5/apache2/php.ini) and at the end of file add:

[zend]
zend_extension=/usr/lib/php5/Zend/ZendGuardLoader.so
zend_loader.enable=1
zend_loader.disable_licensing=0
zend_loader.license_path=

 

After this procedure restart apache using the following commands:
sudo /etc/init.d/apache2 stop
sudo /etc/init.d/apache2 start

 

Installation Manual Scriptcase

 

Download Scriptcase with extension zip or tgz on www.scriptcase.net and extract it in the root of your web environment installed previously (/var/www).

After extracting, rename the folder to scriptcase.

 

Access Scriptcase from your browser followed the example below:

Ex.: http://127.0.0.1/scriptcase

 

The screen below shows a language selection (English) of the ScriptCase installation process.

 

Here he checks the installed extensions. If you tick the checkboxes of the extensions not installed it tries to configure the php.ini file to recognize these extensions.

 

 

 

Now you will choose to enter the Typical Installation or Custom Installation

Typical Installation


 

Scriptcase will install everything automatically. After choosing the typical installation, you will be automatically directed to your Scriptcase environment development. By default, the user is: admin and password: admin.


 

Customized Installation


 

You can choose the database where the Scriptcase will be installed and you can choose the default user and password to access your development environment.

ScriptCase need some permissions to work properly, at this time it checks the permissions.

NOTE: In case of IIS Web Server, permission of Writing, Reading, and Performing For IUSR_NAMEOFSERVER user, the following directories are required:


1 - Directory of the PHP installation.
2 - ScriptCase directory.
3 - Windows cmd.exe file.



This step consists in the definition of the database where the ScriptCase be installed. The default option is SQLite, which requires only that your module is enabled in PHP, but you can install other databases by clicking 'Change Database'. Proceed to create the tables in the database chosen.





The tables used by ScriptCase were created successfully.


 

Here you create a login and password for the ScriptCase administrator.

 

If the installation was successful this screen will appear. Click continue to initialize ScriptCase.

 

Login screen to access the ScriptCase with the username and password previously reported.
