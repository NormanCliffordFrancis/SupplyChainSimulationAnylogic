Connecting Access via ODBC

Home >> Connections >> MS-Access >> Connecting Access via ODBC

The Microsoft Access, also known as MSAccess, is a management system database from Microsoft, packaged with Microsoft Office Professional which combines the Microsoft Database Engine with a graphical user interface.

To create a connection with your ScriptCase and your Access Database using an ODBC, you must create an ODBC Connection in your Windows Panel Control (ODBC must be created on the same server where you installed the ScriptCase).

Entering the ODBC Data Source Administrator, select the ABA Source Data System and create your connection   with your Access Database. After creating the ODBC in Windows Control Panel , you must use the Microsoft Access Database Engine component so there is connection to the Access database.


Installing the Microsoft Access Database Engine component


1. To check whether the 32-bit Microsoft Access Database Engine drivers are installed, go to C:\Windows\SysWOW64\odbcad32.exe. If on a 32-bit machine, go to C:\Windows\System32\odbcad32.exe to access the utility. Go to the drivers tab. If the Microsoft Access Driver lists .mdb and .accdb, then the correct drivers are installed. In this case, click here

2. If Microsoft Access Driver only lists the .mdb, the required driver is not installed.

  • For the 2007 Microsoft Office System (Windows 8, Windows 7, Windows Server 2003, Windows Server 2008, Windows Vista, Windows XP), the 2007 Access Database Engine must be installed.
  • For the 2010 Microsoft Office System (Windows 8, Windows 7, Windows Server 2003 R2 (32-Bit x86), x64 editions of Windows Server 2003 R2, Windows Server 2008 R2, Windows Server 2008 Service Pack 2, Windows Vista Service Pack 1, Windows XP Service Pack 3), the 2010 Access Database Engine must be installed.

    • For more information, see the site of Microsoft Access.

3. After installing the Engine, many new drivers appear in your ODBC Data Source Administrator. The Microsoft Access Driver now list .accdb as a file it supports, as we can see in the images below.


Image1: Before you install Microsoft Access Database Engine (only supports .mdb).


Image: After install Microsoft Access Database Engine (supports .mdb and .accdb).

Now, you can use the following ODBC driver:

 

Accdb Extension - MicroSoft Access Driver (*mdb, *accdb)

To work with MS Access version lower than 2007/2010, click here to see what you have to do.

MDB Extension - MicroSoft Access Driver (*mdb)

 

After creating the ODBC, follow the steps below to create the connection:

 

 

CREATING SCRIPTCASE CONNECTION (MSACCESS)


Now access ScriptCase to create a new connection.


Step 1: Log in your ScriptCase, and on the left menu you can see a tab "Connection", click on the sub item "New".

 


Step 2: After you click "New", will appear at the right screen to create connection, click on the image of the MS Access database.


Step 3: In the picture below, fill in the required specifications:

  • Connection name - Set the name of your new connection.
  • DBMS type - Set the type of the DBMS : ADO or ODBC.
  • Test Connection - Used to test if the connection is successful or not.
  • ODBCName - Enter the ODBC created in the Windows Control Panel.
  • Username - Enter a valid database user name.
  • Password - Enter the valid password.

Para mais detalhes na configuração da conexão do banco da dados clique na aba Advanced

 

  • Decimal separator - Select the type of separator between comma and point.
  • Persistent connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use the schema name before the table - Allows the use of schemas before the names of the tables.
  • Filter - Clicando nessa Aba, aparecerá a seguinte tela:

 

  • Show - Display filters on the tables, views, system tables and procedures.
  • Filters - Define which tables and owners will be displayed.

Step 4: Always test your connection before completing the same, if the message "Connected successfully" is displayed, then complete the establishment of the connection.
