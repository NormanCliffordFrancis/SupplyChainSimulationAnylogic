Enabling Oracle ODBC on Windows - Installer Scriptcase

Home >> Connection >> Oracle >> Oracle ODBC on Windows

 

We will see the steps we enable for Oracle ODBC on Windows, as well as the Oracle connection on the ScriptCase.

 

1 - First we will enter the website and download the Oracle's instant client basic and odbc (www.oracle.com). Choose the 32-bit version because of compatibility of PHP.

 

2 - Extract the folder instant client basic in the root of your computer. (eg.: C:\instantclient_12_1)

 

3 - Let's add on the PATH system the path of this folder:

3.1 - Go to Control Panel > System > Advanced System Settings.
 
3.2 - On the Advanced tab > Click on Environment Variables.
 

 

3.3 - Add the path as shown below:

 

4 - Extract instant client odbc insert and all contained files inside the instant client folder. (eg.: C:\instantclient_12_1\)

4.1 - After copied the files, open the folder C: \ instantclient_12_1 \ and run a file odbc_install.exe

 

5 - Restart the computer so that enable the environment variables.

 

6 - Create a data source system to the driver: Oracle in instantclient_12_1

Obs: The Administrator of the ODBC data source should be initiated with 32 bits.

 

 

Connecting on Scriptcase

 

Open Scriptcase and make a new Oracle connection.

  • Connection Name - Set the name of your new connection.
  • DBMS Type - Version of Oracle used.
  • Test Connection - Displays whether the connection was successful or not.
  • ODBC Name - Should be informed the name of the ODBC administrator created in ODBC data sources from the operating system.
  • Username - Enter the user to grant access to your Oracle.
  • Pasword - Enter the password to grant access to your Oracle.

 

Filter: Clicking on this tab, the following screen appears:


  • Show - Allows the display filter tables, views, system tables and procedures.
  • Searches - Determines which tables and owners will be displayed.

 

For more details on configuring the database connection to the data click on the Advanced tab.

 

  • Client encoding - Encoding for your database.
  • Decimal Separator - Select the type of separator between comma and point.
  • Persistent Connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.
