Enabling Oracle on Mac OSX

Home >> Connection >> Oracle >> Oracle on Mac OSX

 

We will see the steps for us to enable Oracle on Mac OSX, and also the Oracle connection with ScriptCase.

This article was based on the Mavericks distribution using PHP 5.4 manually compiled.

 

1 - First we will enter the Oracle's website and download the Oracle instant client and sdk (www.oracle.com). Choose the 64 bit version because of compatibility of operating system.

 

2 - Create a folder in this directory /usr/local/instantclient/11.2.0.3/ and extract all content of downloaded files:

/usr/local/instantclient/11.2.0.3/

 

3 - Create symbolic links of the system using the following commands:

ln -s /usr/local/instantclient/11.2.0.3/sdk/include/*.h /usr/local/include/
ln -s /usr/local/instantclient/11.2.0.3/*.dylib /usr/local/lib/
ln -s /usr/local/instantclient/11.2.0.3/*.dylib.11.1 /usr/local/lib/
ln -s /usr/local/instantclient/11.2.0.3/libclntsh.dylib.11.1 /usr/local/lib/libclntsh.dylib

 

4 - Now install the Oracle extension for PHP using the following command:

pecl install oci8

 

Note: In case it asks whats the ORACLE_HOME directory, run this command:

instantclient,/usr/local/instantclient/11.2.0.3/

 

5 - After all the steps above, now add on the php.ini the following extent:

extension=oci8.so

 

6 - Restart the web service.

 

7 - Check your diagnosis if your Oracle 8 is enabled.

 

 

Connecting on Scriptcase

 

Open Scriptcase and make a new Oracle connection.

  • Connection Name - Set the name of your new connection.
  • DBMS Type - Version of Oracle used.
  • Test Connection - Displays whether the connection was successful or not.
  • TSNAME - Should be informed the IP of the server where the database is installed and the instance name created in the Oracle installation.
  • Username - Enter the user to grant access to your Oracle.
  • Pasword - Enter the password to grant access to your Oracle.

 

Filter: Clicking on this tab, the following screen appears:


  • Show - Allows the display filter tables, views, system tables and procedures.
  • Searches - Determines which tables and owners will be displayed.

 

For more details on configuring the database connection to the data click on the Advanced tab.

 

  • Client encoding - Encoding for your database.
  • Decimal Separator - Select the type of separator between comma and point.
  • Persistent Connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.
