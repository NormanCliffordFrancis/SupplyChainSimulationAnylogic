Enabling Oracle on Linux

Home >> Connection >> Oracle >> Oracle on Linux

 

We will review the steps for us to enable Oracle on Linux, as well as the Oracle connection in ScriptCase.

This article was based in Debian Wheezy distribution using PHP 5.4 manually compiled.

 

1 - Enter the Oracle's website and download the oracle instant client basic and devel (www.oracle.com). Choose the version according to your operational system and download with .rmp extension.

 

2 - Install the Alien program. It will convert the file from .rmp to .deb, this extension is recognized in debian.

 

3 - Now after the program is installed, run the following command to convert the .rpm file to .deb

 

4 - In this step we will finally install Oracle instant client. Run the following command to perform the installation.

 

5 - Download the extension's installer for Oracle on Linux.

Oracle Extensions - Linux

 

6 - Extract this file and then open the extracted folder.

tar xzf oci8-2.0.8.tgz

cd oci8-2.0.8

 

7 - Now we need to configure PHP with Oracle instant client. After opening the folder oci8-2.0.8, execute the following commands:

./configure

make

make install

 

Note: If after running the command, ./configure, an error occurs, it is due to PHP havent found the path for the instant client. To correct this error only execute the following command and repeat step 7 again.

 

8 - After all the steps above, now add in php.ini the following extent:

extension=oci8.so

 

9 - Restart the web service.

 

10 - Check your diagnosis if your Oracle 8 is enabled.

 

 

Connecting on Scriptcase

 

Open Scriptcase and make a new Oracle connection.

  • Connection Name - Set the name of your new connection.
  • DBMS Type - Version of Oracle used.
  • Test Connection - Displays whether the connection was successful or not.
  • TSNAME - Should be informed the IP of the server where the database is installed and the instance name created in the Oracle installation.
  • Username - Enter the user to grant access to your Oracle.
  • Pasword - Enter the password to grant access to your Oracle.

 

Filter: Clicking on this tab, the following screen appears:


  • Show - Allows the display filter tables, views, system tables and procedures.
  • Searches - Determines which tables and owners will be displayed.

 

For more details on configuring the database connection to the data click on the Advanced tab.

 

  • Client encoding - Encoding for your database.
  • Decimal Separator - Select the type of separator between comma and point.
  • Persistent Connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.
