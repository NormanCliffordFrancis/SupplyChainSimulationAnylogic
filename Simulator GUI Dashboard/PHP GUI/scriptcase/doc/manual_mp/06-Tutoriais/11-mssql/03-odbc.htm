Connecting with MS SQL Server ODBC

 

To connect to the MS SQL Server ODBC you must create a data source system on the same Scriptcase server .

For this go to: Control Panel > System and Security > Administrative Tools > Data Sources ODBC (32 bits)

 

Note: The data source system must be created by "data source administrator ODBC 32-bit", because PHP on Windows is compiled using 32 bit architecture.

 

After creating the data source system, simply create a MS SQL Server ODBC connection on Scriptcase.

 

 

CREATING SCRIPTCASE CONNECTION (MS SQL SERVER ODBC)

    Access ScriptCase and we will create a connection to the database from MSSQL Server ODBC.

Step 1: Access your ScriptCase, create a new project, and on the toolbar go to the highlighted icon in the figure below, referring to the "New Connection".



 

Step 2: After clicking "New" will show up below the screen to create connection, click the image of MSSQL Server database.

 

 

Step 3: In the image below, fill in the required specifications:

  • Connection name - Define a name for your new connection.
  • DBMS Type - Sets whether the type DBMS is ADO, Server, ODBC or Native SRV.
  • Test Connection - Displays whether the connection was successful or not.
  • ODBC Name - Inform the name of your ODBC.
  • Username - Inform the user to get access to your MSSQL.
  • Password - Enter the password to get access to your MSSQL.
  • Filter - Clicking on this tab, the following screen appears:

  • Display - Allows the display filter tables, views, system tables and procedures.
  • Filters - Lets you define which tables and owners will be displayed.

For more details on configuration the connection of database data click on Advanced tab.

 

  • Decinal separator - Select the type of separator between comma and point..
  • Persistent connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.

Step 4: Always test your connection before completing the same. If the message "Connected successfully" appears, complete the establishment of the connection.
If fails to connect, review the fields populated, mainly user name and password ODBC.
