Connecting with MS SQL Server ADO

 

The installer Scriptcase is enabled by default the MS SQL Server ADO extension.

This connection is available for the following operating systems:

  • Operational Systems

    • - Windows XP

    - Windows Server 2003

 

 

Connecting on Scriptcase



Step 1: Access your ScriptCase, and on the toolbar go to the highlighted icon in the figure below, referring to the "New Connection".

 

Step 2: After clicking "New", will appear at the right screen to create connection, click the image of MSSQL Server database.

 

 

Step 3: In the image below, fill in the required specifications:

  • Connection Name - Define a name for your new connection.
  • DBMS Type - Sets whether the model DBMS is ADO, Server, ODBC or Native SRV.
  • Test Connection - Displays whether the connection was successful or not.
  • Server/Host (Name or IP) - Enter the Name/IP of your MS SQL Server.
  • Database - Enter the database name of your database.
  • Username - Inform the user to get access to your MSSQL.
  • Password - Enter the password to get access to your MSSQL.
  • Filter - Clicking on this tab, the following screen appears:

  • Display - Allows the display filter tables, views, system tables and procedures.
  • Filters - Lets you define which tables and owners will be displayed.

For more details on configuration the connection of database data click on Advanced tab.

 

  • Decinal separator - Select the type of separator between comma and point..
  • Persistent connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.
