Connecting with MS SQL Server Native SRV - Windows

 

To enable the extension MS SQL Server Native SRV is necessary to install the Native Client 2012.

 

The following operating systems are recommended:

  • Operational Systems

    • - Windows Server 2012

    - Windows 8

    - Windows Server 2008

    - Windows 7

     

  • Download of SQL Server Native Client

    • Native Client 64 bits

    Native Client 32 bits

 

Below is the step-by-step to install Native Client SQL Server 2012 and connect to Scriptcase.

 

Installing SQL Server Native Client 2012

 

After downloading the file, run the installer and follow the steps below:

 

Accept the license agreement and click Next.

 

Allow this option checked the option "Client Components" .

Note: No need to check the option "Sql Server Native Client SDK", because the components you'll need is in the first option..

 

Click Install to begin installing the client.

 

The program features you selected are being installed.

 

Click Finish to exit.

 

 

Connecting on Scriptcase

 

To create the connection, choose the model DBMS: MSSQL Server NATIVE SRV

 

  • Server/Host (Name or IP): Enter the IP from server is located where the MS SQL Server.
  • Username: Enter the User Database.
  • Password: Enter the Password Database.

  • Database: Click "List database" and select the required database.
  • Filter - Clicking on this tab, the following screen appears:

  • Display - Allows the display filter tables, views, system tables and procedures.
  • Filters - Lets you define which tables and owners will be displayed.

For more details on configuration the connection of database data click on Advanced tab.

 

  • Decinal separator - Select the type of separator between comma and point..
  • Persistent connection - Persistent connections are links that do not close when the execution of your script ends.
  • Use schema before the table name - Allows the use of schemas before the names of the tables.

To end the connection, click "Save".
